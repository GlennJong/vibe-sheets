Vibe Sheets Privacy Policy & Google OAuth Scopes

Last Updated: January 7, 2026

Introduction

Vibe Sheets is a client-side development tool designed to help you quickly create simple databases using Google Sheets. To achieve this, we require your authorization for specific Google API scopes via Google's OAuth 2.0 mechanism. This document details the permissions we request and their purpose.

Permissions (Scopes) We Request and Their Usage

When you grant access, Vibe Sheets requests the following Google API scopes:

  • https://www.googleapis.com/auth/spreadsheets

    Purpose: To create and modify Google Spreadsheets. This is the core function of the tool, used to generate new spreadsheet files on your behalf.

  • https://www.googleapis.com/auth/script.projects

    Purpose: To create and modify Google Apps Script projects. We need this permission to generate a script bound to your spreadsheet, which provides the API functionality.

  • https://www.googleapis.com/auth/script.deployments

    Purpose: To create and manage deployments for Apps Script. This allows us to deploy the generated script as a web app, enabling it to return JSON data.

  • https://www.googleapis.com/auth/drive.file

    Purpose: To view, create, and modify specific Google Drive files you open or create with this app. We use this to search for the spreadsheets and scripts you've created.

  • https://www.googleapis.com/auth/drive.metadata.readonly

    Purpose: To view metadata for files in your Google Drive (e.g., file name and description). We use this to find and store the Web App URL for subsequent connection tests.

Data Storage and Security

The core design philosophy of Vibe Sheets is decentralization and user control. We emphasize the following:

  • Client-Side Operation: This application runs entirely within your browser. No data is sent to our servers, other than direct communication with Google's APIs.
  • No Data Storage: We do not store your personal information, Google account data, or the content of any files you generate.
  • Token Security: Your Google OAuth Access Token is stored only in your browser's memory for the current session. It is discarded when you close the tab.

Vibe Sheets 隱私權與 Google OAuth 權限說明

最後更新日期：2026年1月7日

前言

Vibe Sheets 是一個純前端開發工具，旨在協助您快速建立以 Google Sheets 為基礎的簡易資料庫。為了實現此功能，我們需要透過 Google 的 OAuth 2.0 機制，請求您授權特定的 Google API 權限。本文件將詳細說明我們請求的權限以及其用途。

我們請求的權限與用途

在您授權登入時，Vibe Sheets 會請求以下 Google API 權限：

  • https://www.googleapis.com/auth/spreadsheets

    用途：建立與修改 Google 試算表。這是本工具的核心功能，用於為您產生新的試算表檔案。

  • https://www.googleapis.com/auth/script.projects

    用途：建立與修改 Google Apps Script 專案。我們需要此權限來產生一個與您的試算表綁定的腳本，以提供 API 功能。

  • https://www.googleapis.com/auth/script.deployments

    用途：建立與管理 Apps Script 的部署。此權限讓我們能將產生的腳本部署為一個可公開存取的 Web App，使其能回傳 JSON 資料。

  • https://www.googleapis.com/auth/drive.file

    用途：讀取、建立與修改您透過此應用程式開啟或建立的特定 Google Drive 檔案。我們用它來搜尋您已建立的試算表與腳本。

  • https://www.googleapis.com/auth/drive.metadata.readonly

    用途：讀取您 Google Drive 中檔案的元資料（例如檔案名稱與描述）。我們利用此權限來尋找並儲存 Web App 的 URL，以便後續連線測試。

資料儲存與安全性

Vibe Sheets 的核心設計理念是去中心化與使用者自主。我們強調以下幾點：

  • 純前端運作：本應用程式完全在您的瀏覽器中執行。除了與 Google API 的直接通訊外，沒有任何資料會被傳送到我們的伺服器。
  • 無資料儲存：我們不會儲存您的個人資訊、Google 帳號資料，或是您產生的任何檔案內容。
  • Token 安全性：您的 Google OAuth Access Token 僅儲存在瀏覽器的記憶體中，用於當前的連線階段。關閉分頁後，Token 即會消失。
