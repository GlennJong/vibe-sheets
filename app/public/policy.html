Vibe Sheets Privacy Policy & Google OAuth Scopes

Last Updated: January 13, 2026

Introduction

Vibe Sheets is a client-side development tool designed to help you quickly create simple databases using Google Sheets. To achieve this, we require your authorization for specific Google API scopes via Google's OAuth 2.0 mechanism. This document details the permissions we request and their purpose.

Permissions (Scopes) We Request and Their Usage

When you grant access, Vibe Sheets requests the following Google API scopes:

  • https://www.googleapis.com/auth/script.projects

    Purpose: To create and modify Google Apps Script projects. We need this permission to generate a script bound to your spreadsheet, which provides the API functionality.

  • https://www.googleapis.com/auth/script.deployments

    Purpose: To create and manage deployments for Apps Script. This allows us to deploy the generated script as a web app, enabling it to return JSON data.

  • https://www.googleapis.com/auth/drive.file

    Purpose: To view, create, and modify specific Google Drive files you open or create with this app. We use this to search for the spreadsheets and scripts you've created.

How We Access and Use Your Data

To provide its services, Vibe Sheets interacts with Google APIs. Below is a clear breakdown of what data we access and how we use it.

Data We Access

Vibe Sheets accesses the following types of data from your Google Account:

  • Google Spreadsheets: We access spreadsheets that you create through our application. This includes the ability to create new sheets, and read/write data to them.
  • Google Apps Script Projects: We create Apps Script projects that are bound to the spreadsheets you create. This includes the script's code and its properties.
  • Google Drive File Metadata: We access metadata for files in your Google Drive, such as file names and IDs, to locate the spreadsheets and scripts created by the app.

How We Use Your Data

Your data is used exclusively to enable the core functionality of Vibe Sheets:

  • To Create Spreadsheets: The primary function of our app is to generate a new Google Sheet that acts as a simple database.
  • To Automate APIs: We create and deploy a Google Apps Script, which turns your spreadsheet into a web-based API that can return JSON data.
  • To Locate and Manage Files: We use Drive API access to find the files created by the app so you can manage them easily.

Data Storage and Security

The core design philosophy of Vibe Sheets is decentralization and user control. We emphasize the following:

  • Client-Side Operation: This application runs entirely within your browser. No data is sent to our servers, other than direct communication with Google's APIs.
  • No Data Storage: We do not store your personal information, Google account data, or the content of any files you generate.
  • Token Security: Your Google OAuth Access Token is stored only in your browser's memory for the current session. It is discarded when you close the tab.

Data Sharing and Disclosure

Vibe Sheets allows you to use your Google User Data. We do not share, transfer, or disclose your Google user data with any third parties. Vibe Sheets is a client-side tool, and your data remains within your Google account and browser session.

Data Protection Mechanisms

We implement the following data protection mechanisms for your sensitive data:

  • Encryption in Transit: All communication between the application and Google APIs occurs over secure HTTPS connections.
  • No Persistent Storage: Authentication tokens are stored in-memory and are cleared when the session ends. We do not store any of your data on our servers.

Data Retention and Deletion

This application does not retain any user data. All spreadsheets and scripts created are stored directly in your Google Drive. You retain full ownership and control over this data. You can delete any created files at any time via the Google Drive interface.

Contact Us

If you have any questions about this Privacy Policy, please feel free to open an issue on our GitHub project page.

Vibe Sheets 隱私權與 Google OAuth 權限說明

最後更新日期：2026年1月13日

前言

Vibe Sheets 是一個純前端開發工具，旨在協助您快速建立以 Google Sheets 為基礎的簡易資料庫。為了實現此功能，我們需要透過 Google 的 OAuth 2.0 機制，請求您授權特定的 Google API 權限。本文件將詳細說明我們請求的權限以及其用途。

我們請求的權限與用途

在您授權登入時，Vibe Sheets 會請求以下 Google API 權限：

  • https://www.googleapis.com/auth/script.projects

    用途：建立與修改 Google Apps Script 專案。我們需要此權限來產生一個與您的試算表綁定的腳本，以提供 API 功能。

  • https://www.googleapis.com/auth/script.deployments

    用途：建立與管理 Apps Script 的部署。此權限讓我們能將產生的腳本部署為一個可公開存取的 Web App，使其能回傳 JSON 資料。

  • https://www.googleapis.com/auth/drive.file

    用途：讀取、建立與修改您透過此應用程式開啟或建立的特定 Google Drive 檔案。我們用它來搜尋您已建立的試算表與腳本。

我們如何存取與使用您的資料

為了提供服務，Vibe Sheets 會與 Google API 互動。以下清楚說明我們存取的資料以及其使用方式。

我們存取的資料

Vibe Sheets 會從您的 Google 帳戶存取以下類型的資料：

  • Google 試算表： 我們會存取您透過本應用程式建立的試算表。這包括建立新試算表以及讀寫其中資料的能力。
  • Google Apps Script 專案： 我們會建立與您所建試算表綁定的 Apps Script 專案。這包括腳本的程式碼及其屬性。
  • Google Drive 檔案元資料： 我們會存取您 Google Drive 中檔案的元資料（例如檔案名稱和 ID），以定位由本應用程式建立的試算表和腳本。

我們如何使用您的資料

您的資料僅用於實現 Vibe Sheets 的核心功能：

  • 建立試算表： 本應用的主要功能是產生一個可作為簡易資料庫的 Google 試算表。
  • 自動化 API： 我們會建立並部署一個 Google Apps Script，將您的試算表轉換為一個可回傳 JSON 資料的網路 API。
  • 定位與管理檔案： 我們使用 Drive API 權限來尋找由本應用程式建立的檔案，以便您輕鬆管理它們。

資料儲存與安全性

Vibe Sheets 的核心設計理念是去中心化與使用者自主。我們強調以下幾點：

  • 純前端運作：本應用程式完全在您的瀏覽器中執行。除了與 Google API 的直接通訊外，沒有任何資料會被傳送到我們的伺服器。
  • 無資料儲存：我們不會儲存您的個人資訊、Google 帳號資料，或是您產生的任何檔案內容。
  • Token 安全性：您的 Google OAuth Access Token 僅儲存在瀏覽器的記憶體中，用於當前的連線階段。關閉分頁後，Token 即會消失。

資料分享與揭露

Vibe Sheets 允許您使用您的 Google 使用者資料。我們不會與任何第三方分享、轉讓或揭露您的 Google 使用者資料。Vibe Sheets 是一個使用者端工具，您的資料僅保留在您的 Google 帳戶和瀏覽器工作階段中。

資料保護機制

我們針對您的敏感資料實施以下保護機制：

  • 傳輸加密：應用程式與 Google API 之間的所有通訊皆透過安全的 HTTPS 連線進行。
  • 無持久儲存：驗證權杖僅儲存在記憶體中，並在工作階段結束時清除。我們不會在我們的伺服器上儲存您的任何資料。

資料保留與刪除

本應用程式不會保留任何使用者資料。所有建立的試算表和腳本皆直接儲存在您的 Google Drive 中。您擁有這些資料的完整所有權與控制權。您可以隨時透過 Google Drive 介面刪除任何已建立的檔案。
